Exercise 34: Using more descriptive variable names in your function definitions to make your code easier to understand.

NEW: Using multiple function definitions, and function calls with better variable names; 

EXAMPLE: In the function definition blinkOnce below - you can see that the integer t is used to control the delay time and the integer n is used to control the LED that is turned on / off. This all works as expected.

 

void blinkOnce(int t, int n)
{
 digitalWrite(n, HIGH);
  delay(t);
  digitalWrite(n, LOW);
  delay(t);
} 

 

However: If we instead use more descriptive variable names in the function definition the resulting code is much easier to read and understand.

 

void blinkOnce(int delayTime, int ledNumber)
{
 digitalWrite(ledNumber, HIGH);
  delay(delayTime);
  digitalWrite(ledNumber, LOW);
  delay(delayTime);
}

Now the variable names (delayTime and ledNumber) describe what the variables are used for, which makes the code easier to read and understand.
