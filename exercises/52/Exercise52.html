Exercise 52: Learning how to use If/else statements to make decisions inside your code based on variables and other conditions.

 

NEW: Using If/else in the Arduino programming environment to to make decisions;

 

DESCRIPTION:

 

In life, sometimes you need to make a decision. Do you turn left or right there? Do you take the front seat or the back seat?

 

Likewise, sometimes computer programs will have to make decisions. Examples:

On a car, suppose it detects it is dark outside. Should the headlights be on or off?

On any computer system with a login, suppose someone enters the correct password. Do we let them log in?

 

If you want to make some rules to describe these things in English, you will probably use the word ‘if’. Like so:

 

If it is dark outside, then turn the headlights on.

If the user inputs the correct password, then log them in.

 

This works. But you might also want to say what happens if the conditions are not met. Like so:

 

If it is dark outside, turn the headlights on. Otherwise, turn them off. (Maybe the sun just came up, we don’t want the headlights to stay stuck on).

If the user inputs the correct password, log them in. For anything else, print “ACCESS DENIED” in all caps and make an obnoxious noise.

 

You can pretty much make the same kinds of statements in C.

 

EXAMPLE:

 

int x=7;

if (x==4)

{

digitalWrite(5,HIGH);

}

 

Before you try to deduce what this does, you should probably know what that ‘==‘ symbol inside the ‘if’ statement is. That tests for equality. For instance, 3==3 is true, 4==3 is false. We use a double equal sign instead of one equal sign to tell the difference between assignment and comparison. For instance:

 

x=3 says ‘x gets the value of 3’

x==3 says ‘is x equal to 3?’

 

Therefore, this if statement makes the LED at pin 5 turn on if x is equal to 4. Since x is 7, this will do nothing.

 

There’s a ridiculously common bug out there where programmers confuse the two equal signs and write stuff like if(x=3) when they meant if(x==3). Don’t do that. If you really want to know what happens when using the wrong equal sign, here’s an example:

 

int x=5;

if(x=3)

{

Serial.print(“x is equal to 3!”);

}

 

The concept of ‘true’ in an if statement is represented by any non-zero number (almost always 1). False is equal to 0. When you write something like ‘x==5’, the whole statement evaluates to 1 if x is 5, and 0 otherwise. The statement ‘x=3’ evaluates to 3 (yes it’s weird). 3 is non-zero, and therefore represents true.

So what actually happens? No matter what x is, it will print out “x is equal to 3!”. So saying this: if(x=3){… is pretty much saying “did x get assigned something that isn’t zero?”

 

We’ve seen ‘==‘ so far, but there are a bunch of other symbols that can be used for comparison:

“Else”

The next major feature is the ‘else’ keyword. This is how you would say ‘otherwise’ with an if statement. Example:

 

if(x < 10)

{

Serial.print(“x is less than 10”);

}

else

{

Serial.print(“x is not less than 10”);

}

 

Another feature is “else if”. Maybe you don’t want one big “otherwise” statement but different courses of action to take. This example converts the numbers 0-2 into English representations:

 

if(x==2)

{

Serial.print(“two”);

}

else if(x==1)

{

Serial.print(“one”);

}

else if(x==0)

{

Serial.print(“zero”);

}

else

{

Serial.print(“x is not 0,1, or 2”);

}

 

Each if statement gets tested top to bottom and exactly one statement executes. If you do something like this, the second statement will never be hit:

 

int x=4;

int y;

if(x==4)

{

y=9000;

}

else if(x==4)

{

y=9001; //This will never execute

}

 

Fancier Expressions

So maybe you want to get fancy with your expressions. Suppose you want to test that a number is over 9000 AND less than 10000. Let that number be our favorite variable ‘x’. This is how you would do it:

 

if(x>9000 && x<10000)

{

//x is between 9000 and 10000

}

 

That ‘&&’ symbol means ‘AND’, but there are a few more you can use. Here are all of them:

Symbol

What it means

Example true statement

And now some Examples.

 

This tests if x is not equal to 10:

 

if( !(x==10) )

{

//x is is not equal to 10

}

 

But you could also just say x!=10, like so:

 

if( x!=10 )

{

         //x is is not equal to 10

}

 

Test if x is less than -50 or greater than 50:

 

if(x<-50 || x>50)

{

// x is less than -50 or greater than 50:

}

 

The same thing, except this time saying ‘x is NOT between -50 and 50’:

 

if( !(x>=-50 && x<=50) )

{

// x is NOT between -50 and 50

}

 

Note how we have ‘greater than or equal to’ instead of just ‘greater than’. Remember that ‘not less than -50’ is the same as ‘greater than or equal to -50’.  

 

You can also combine AND and OR to do some interesting things:

if( x>=-10 && x<=-5 || x>=0 && x <= 5)

{

// x is in the [-10,-5] range or the [0,5] range

}

This checks if x is in the [-10,-5] range or the [0,5] range.

 

Remember your order of operations? This programming language has that too. As it turns out, ‘&&’ comes before ‘||’ in the order of operations. So something like this:

x>10 && x<20 || x>100

Is the same as saying this:

(x>10 && x<20) || x>100

And just like in your algebra class, whatever’s in the parentheses gets executed first.
