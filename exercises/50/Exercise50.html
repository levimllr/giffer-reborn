Exercise 50: Learning how to use the serial monitor to look inside and see what’s happening inside our program as it is running.

 

NEW: Serial.print() in the Arduino programming environment;

 

DESCRIPTION: Serial.print() uses pins 0 and 1 via the USB cable to send messages back to the computer (whenever the Arduino is actually connected to a computer.)   (That, by the way, is another reason we can’t use pins 0 and 1 for LEDs.) So how do we do this?  First, let’s look at a working example, and then break down the code:

 

EXAMPLE:

void setup()

{

   Serial.begin(9600);

}

 

void loop()

{

   Serial.print(“Hello World!”);

}

 

The first new thing we see in this code is the line “Serial.begin(9600);”  This line goes a little beyond the scope of this course, however it is necessary to include it in your programs, otherwise print statements will not work.  It is essentially setting the rate of data transfer between the computer and Arduino to be 9600 bits/sec.

 

The second new thing we see is the actual “Serial.print()” statement, with “Hello World!” inside the parenthesis.  You can think of this as “Serial.print(_______)” with the ___ signifying an empty space to fill in with what you want to be printed to the screen.  Thus, since “Hello World!” is filling in the blank, that is what will be printed to the screen. Now you might be thinking, “Where exactly does this print to the screen?”  To see that, we must open the serial monitor as shown below:

 

Clicking the little magnifying glass in the top right opens the “COM” window, which is where any print statements in your code will print to.  You must make sure that the dropdown box in the lower right of the COM window says 9600 baud, as this pairs with our Serial.begin(9600) to set the correct data transfer rate.   Also, make sure you capitalize the ‘S’ in “Serial” -- it is case sensitive! As we can see, since our print statement is inside the loop, our COM window displays “Hello World!” and then prints and displays again infinitely.

 

Another thing to know is that Giffer knows how to deal with Serial.print statements and how to check them for accuracy. You can see the serial output of your code in Giffer by clicking on the Output Console tab.

 

NEW: Using Serial.println() in the Arduino programming environment to print to a new line;

 

DESCRIPTION:

A close relative to the print statement is the print newline statement Serial.println()

 

The print newline statement is identical to the print statement in every way, except that it appends a newline at the end of what you print.  This is most obvious by looking at the example below:

 

EXAMPLE:

 

void setup()

{

   Serial.begin(9600);

}

 

void loop()

{

   Serial.print(“hi”);

   Serial.print(“bye”);

}

 

If we look on the serial port monitor we will see the following output

 

hibye

 

Compare this to the next example:

 

void setup()

{

   Serial.begin(9600);

}

 

void loop()

{

   Serial.println(“hi”);

   Serial.println(“bye”);

}

 

If we look on the serial port monitor we will see the following output

 

hi

bye

 

Notice that println appended a newline, and thus bye is printed on the line below hi, whereas the simple print statement adds the next print alongside the original print, as seen in the example.  This is sometimes useful to improve readability when printing. Another useful trick is to use Serial.println(“”); which will simply print a blank line to the COM, adding more whitespace to improve readability.
