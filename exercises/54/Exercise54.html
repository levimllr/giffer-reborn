Exercise 54: Learning how to use the Knob / Potentiometer on the Light Sculpture board

 

NEW: Reading the On / Off state of the knob / potentiometer on the Light Sculpture board

 

DESCRIPTION:

 

By now you’re probably wondering what that knob on the board is for. That’s called a potentiometer. You turn it, and the Arduino can read what position it is in. We’d like to change our program’s behavior based on what the potentiometer is doing. For this, we need to make some decisions. Perfect job for if/else statements.

 

First, we need to know what pins the knob / potentiometer can change. The potentiometer is connectected to two pins on the arduino - similarly to how the LED’s are connected to pins on the Arduino. However - with the pins connected to the potentiometer the pins are inputs on the Arduino as opposed to the LED’s which are outputs on the Arduino.

 

Input / Output pins - what’s the difference?

 

Output pins, we have been using them to turn on and off the LEDs in our light sculpture board - Send information to the light sculpture board.

 

Input pins,  we will be using them to get information from the light sculpture board. Getting information from the user using the light sculpture board.

 

For this first exercise using the potentiometer, we will only be focusing on the first pin connected to the potentiometer. Pin 18 on the Arduino is connected to the potentiometer like an ON / OFF switch. If you rotate the Knob from all the way clockwise to all the way counterclockwise you will hear a click at the end of its counterclockwise rotation. This click is connecting Pin 18 on the Arduino to ground. If we rotate the Knob clockwise you will the click again and that means that Pin 18 is no longer connected to ground.

 

When Pin 18 is connected to ground - if we read its value from the Arduino Pin 18 = 0.

 

When Pin 18 is not connected to ground - if we read its value from the Arduino Pin 18 = 1.

 

Second, we need to know how to read in a value from the potentiometer. Fortunately, it’s easier than you think. This line does it all:

 

int pin18 = digitalRead(18);

 

digitalRead() is a function that can read values from pins on the Arduino. Just like pinMode() or digitalWrite(), digitalRead() needs a pin to use.

 

int pin18 = digitalRead(18); // This line sets the variable “pin18” equal to the state pin 18 is in.

 

In summary:

 

When the knob is rotated all the way counterclockwise, Pin 18 is connected to ground. If we read its value from the Arduino Pin 18 = 0. If we run the next line of code with the knob in this state, the variable pin18 will be equal to 0.

pin18 = digitalRead(18);

 

When the knob is rotated in any other state other than all the way counterclockwise, Pin 18 is connected to five volts. If we read its value from the Arduino Pin 18 = 1. If we run the next line of code with the knob in this state, the variable pin18 will be equal to 1.

pin18 = digitalRead(18);



EXAMPLE:

 

void setup()

{

 Serial.begin(9600); // Setup the serial port speed

 pinMode(18, INPUT);

}

 

void loop()

{

 int pin18 = 0;     // variable to store the value of pin 18

 pin18 = digitalRead(18); // read the input pin either HIGH or LOW

 

 if (pin18 == 0)

 {

   Serial.println("Pin 18 is equal to zero");

 }

 if (pin18 == 1)

 {

   Serial.println("Pin 18 is equal to one");

 }

 delay(500);

}

 

If we run this code on a Arduino with a light sculpture connected and look at the serial output we will see the output change based on the position of the knob.

 

Giffer and the Potentiometer

We can use Giffer to read the state of the potentiometer, but first you have to setup Giffer to give it a timeline of the values of the potentiometer. This can be done by clicking on the Configure Board tab at upper right on the giffer page. Feel free to check it out now. Note - that to make giffer interpret a pin as being one (1) - you need to give it a value of 1023. This isn’t a bug - rather it allows us to use giffer to interpret analog pins as well - we will go over that further in the next exercise.

 

For exercises 54 - 56, giffer will be setup with specific timelines for the inputs that are connected to the potentiometer, which I will describe in the Activity directions.
